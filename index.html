Type safe

Zangle is type safe, all blocks within are strongly typed acording to the given language tag making block merges of different types a compile-time error rather than a documentation bug.

```{.html #example}
<code>
<pre>
{{zig-html-test|escape html}}
</pre>
</code>
```

error on line 4 col 3: `zig' does not match the expected type `html'
{{zig-html-test|escape html}}
  ^~~~~~~~~~~~~

Merging between blocks of different types is still allowed with safe casts using from(type) which ensures the specified type of the block matches the type specification.

```{.html #example}
<code>
<pre>
{{zig-html-test:from(zig)|escape html}}
</pre>
</code>
```

Filters (incomplete)

Zangle supports both inline and external filters on placeholders which enable running additional tools over the tangled code block before it's written to the document.

```

    return text
```

Declare a function for generating html.

```{.zig #example-code-block}
<<imports>>

pub fn main() !void {
    const text = try cwd().readFileAlloc(
        gpa,
        "filters.md",
    );
    defer gpa.free(text);

    var tree = try parse(gpa, text, .{});
    defer tree.deinit();

    try stdout.writeAll("<ul>");

    var it = tree.query(.filename);
    while (it.next()) |filename| {
        try stdout.print(
            \\<li>{s}</li>
        , .{filename});
    }

    try stdout.writeAll("</ul>");
}
```
